# v0.7 Private Explainable Voice Pipeline

v0.7 makes the production path a private, auditable voice-agent stack. Native V2V models remain research candidates, but the runtime target is an explainable cascade whose clinical behavior is deterministic.

## Production Stack

```text
patient audio
  -> browser audio capture with echo cancellation/noise suppression
  -> local STT: SenseVoiceSmall through FunASR
  -> local structured extractor: Qwen-compatible local LLM
  -> deterministic OA ConversationEngine
  -> constrained response wording
  -> local TTS: Qwen3-TTS, then browser TTS fallback
```

## Model Roles

| Layer | v0.7 model/tool | Role | Clinical authority |
|---|---|---|---|
| Audio capture | Browser MediaRecorder/WebRTC audio constraints | Capture patient turns privately | No |
| VAD/endpointing | Manual turn button in v0.7 | Bound each answer turn | No |
| STT | SenseVoiceSmall via FunASR | Convert speech to transcript | No |
| Structured extraction | Local Qwen-compatible model | Extract identity, pain, side-effect, and safety slots | No |
| Clinical control | `ConversationEngine` | Decide next step, validation, red flags, report | Yes |
| Verbalizer | Deterministic required line, optional local rewrite | Make required prompt speakable | No |
| TTS | Qwen3-TTS local, browser fallback | Speak assistant response | No |

## Runtime Endpoints

| Endpoint | Purpose |
|---|---|
| `GET /api/voice_status` | Reports private pipeline, local AI, STT, and TTS readiness |
| `POST /api/private/start` | Starts a deterministic OA session for the private voice pipeline |
| `POST /api/private/turn` | Accepts one recorded patient turn and optional browser transcript fallback |
| `POST /api/message` | Typed deterministic fallback |
| `POST /api/tts` | Local/server speech synthesis |

## Streaming Roadmap

v0.7 records one answer turn at a time, but the service contracts are designed for streaming upgrades.

| Boundary | v0.7 behavior | Later target |
|---|---|---|
| Audio input | Browser records one turn | WebRTC frame streaming |
| Endpointing | User presses start/stop answer | WebRTC VAD or Silero VAD |
| STT | Full turn transcription | Partial transcripts with final commit |
| Extractor | Final transcript only | Speculative extraction from partial text |
| Rulebook | Final transcript only | Final transcript only, for safety |
| TTS | Whole-line synthesis | Streaming TTS and phrase caching |
| Barge-in | Stop recording/playback manually | Automatic user-speech interruption |

Partial transcripts must not update clinical state. They may only be used for captions, prefetch, or speculative preparation.

## Latency Metrics

`/api/private/turn` returns per-turn timing metadata:

- `stt_ms`
- `clinical_engine_ms`
- `total_ms`

Future versions should add endpointing delay, extraction delay, verbalizer delay, TTS first-audio delay, and playback cancellation time.

## Research Side Notes

### Covo

Covo remains a half-duplex V2V research experiment. It is not the v0.7 production path because the open release does not provide the full-duplex callbot runtime expected by stakeholders.

### GLM-4-Voice

GLM-4-Voice remains a research candidate for a native speech-to-speech interface to the same rulebook. It is not the v0.7 production path because structured tool behavior, clinical auditability, and safe fine-tuning need separate evaluation.
