# v0.9.4

Date: 2026-07-22

Status: Implemented baseline. Scope may still change; append later updates below rather than replacing this record.

## Goal

Redesign the voice-call UI so the conversation history can continue auto-scrolling without pushing the live speaking/listening controls and visual cue out of view.

## Implemented Changes

- Split the call panel into a scroll-contained transcript history and a sticky live voice dock.
- Moved the current live prompt, voice orb, mic meter, turn timer, and start/stop controls into the persistent dock.
- Kept transcript auto-scroll, but changed it to scroll only the transcript container instead of calling `scrollIntoView` on the newest message.
- Added desktop viewport containment so the assistant panel and handoff panel scroll independently.
- Added mobile sizing so the voice dock remains visible at the bottom of the call surface while conversation history scrolls above it.
- Updated visible v0.9.4 strings, HTML title, report metadata, and environment checker.

## UX Rationale

Popular chat and live-call interfaces separate history from live interaction state. The history can scroll, but the current interaction affordance stays pinned. v0.9.4 applies that pattern to OA-VA: transcript above, live answer state below.

## Known Limitations

- This is still the half-duplex private voice UI. It improves visibility and turn-taking cues, but does not by itself create true streaming full-duplex.
- Final visual tuning should be based on another real-device run because browser chrome and microphone permission UI can affect available mobile viewport height.
