# v0.8.1

Date: 2026-07-09

## Goal

Polish the voice interaction layer introduced in v0.8.0 so stakeholders can see when the app is listening, recording, processing, and ready for the next patient answer.

## User-Facing Changes

- Added a live voice activity meter to the call surface.
- Added a per-turn recording timer with tabular numerals.
- Added a short turn-status line that distinguishes idle, recording, and processing states.
- Made the stop-recording button visually distinct while recording.
- Added clearer keyboard focus states for primary controls, backup input, and language selection.
- Improved cleanup of voice-state visuals when a turn ends, an error occurs, or a call is reset.

## Backend Changes

- None. This remains a frontend-only interaction polish update.

## Safety/Clinical-Rule Changes

- None. The deterministic OA clinical engine and report generation are unchanged.

## Tests Run

- JavaScript syntax check.
- Python smoke test for the existing clinical flow.

## Known Limitations

- The meter uses Web Audio microphone analysis when a stream is available; browser speech-recognition fallback shows a static active meter because the browser API does not expose the raw microphone stream.
- The meter is not persisted into reports and does not affect clinical decisions.
