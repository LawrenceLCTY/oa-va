# v0.2.1 Version Log

Date: 2026-06-18

## Goal

Improve the autonomous bilingual call flow using selected ideas from `sara_prd.txt` without changing the product into a researcher-operated dashboard.

## Product Decision

v0.2.1 keeps the app as an autonomous patient-facing call assistant.

Not adopted:

- researcher-operated interview dashboard
- single-file HTML architecture
- localStorage-only storage
- removal of backend validation
- manual-only data entry as the primary workflow

Adopted:

- call readiness checks
- permission to continue
- caregiver/respondent source
- both 24-hour average pain and current pain
- expanded side-effect details
- researcher alert flag for follow-up
- explicit research prototype disclaimer

## User-Facing Changes

- Opening flow now includes research prototype disclaimer.
- The assistant asks:
  - whether the participant can hear clearly
  - whether now is a suitable time
  - whether it may continue
- The assistant records whether responses are from:
  - participant independently
  - participant with caregiver assistance
  - caregiver proxy
  - unknown
- The assistant asks for:
  - average pain over the past 24 hours
  - current pain right now
- The UI displays a research prototype disclaimer.

## Side-Effect Detail Expansion

If the participant reports side effects or is uncertain, the assistant now asks:

- symptom description
- symptom start time
- ongoing/resolved/unknown
- mild/moderate/severe/unknown
- medication reduced/paused/stopped
- doctor contacted
- emergency visit or hospitalization

The report labels symptoms as participant-reported and does not state medication causality.

## Researcher Alert Logic

The report now includes `researcher_alert_required`.

This is set when:

- symptom severity is severe
- medication was reduced, paused, or stopped
- emergency visit or hospitalization occurred

Urgent red flags still trigger patient-facing urgent-care advice.

## Known Limitations

- Identity still uses name/mobile/age by product decision; study ID is not implemented yet.
- Side-effect detail collection is still rule-based.
- Researcher alert is recorded in the report but does not yet trigger an external notification.
- Caregiver/proxy source is captured but does not yet alter validation strictness.
- No persistent database or escalation queue is implemented.
