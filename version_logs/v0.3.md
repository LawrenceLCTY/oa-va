# v0.3 Version Log

Date: 2026-06-18

## Goal

Add local/private model support to make the assistant sound more natural while preserving deterministic clinical safety behavior and JSON reports.

## User-Facing Changes

- Assistant speech can request backend-generated audio through `/api/tts`.
- Local TTS uses Kokoro `hexgrad/Kokoro-82M-v1.1-zh`.
- Startup now returns and speaks all initial assistant messages instead of only the last one.

## Backend Changes

- Added `app/tts.py`.
  - Uses Kokoro `KPipeline`.
  - Uses `lang_code="z"` and voice `zf_001` for Chinese.
  - Uses `lang_code="a"` and voice `af_maple` for English.
  - Caches generated WAV audio in a temp cache directory.
- Added `/api/tts`.
- Added `/api/voice_status`.
- Added `app/qwen_server.py`, a lightweight OpenAI-compatible Qwen server for local testing.

## LLM Behavior

- The existing `LOCAL_LLM_URL` hook remains the app integration point.
- Qwen polishing is constrained to humanize wording only.
- Red-flag detection, escalation, validation, and JSON report generation remain deterministic.

## Files Changed

- `app/llm.py`
- `app/main.py`
- `app/qwen_server.py`
- `app/tts.py`
- `static/app.js`
- `README.md`
- `version_logs/README.md`

## Tests Run

- `python3 tests/smoke_test.py`

## Known Limitations

- Kokoro dependencies and model files must be installed/downloaded before `/api/tts` can synthesize audio.
- The included Qwen server is lightweight for local prototyping, not a production process manager.
