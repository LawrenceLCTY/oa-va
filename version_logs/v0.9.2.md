# v0.9.2

Date: 2026-07-22

Status: Implemented baseline. Scope may still change; append later updates below rather than replacing this record.

## Goal

Make the v0.9.x questionnaire flow reliable enough for another stakeholder run by fixing the failure modes exposed by `reports/oa-report-2026-07-22T18-59-18-aca22e42.json` and aligning execution with the DOCX questionnaire plus sample phone transcript.

## Failure Analysis Inputs

- Bad run report: `reports/oa-report-2026-07-22T18-59-18-aca22e42.json`
- Questionnaire source: `data/OA问卷-房山-简化-v2.docx`
- Target conversation style: `data/新录音 4.txt`

## Implemented Changes

- Added `app/version.py` and centralized report/pipeline metadata around v0.9.2.
- Added STT transcript sanitation for SenseVoice metadata tags and content-free output.
- Rejected unusable local STT output without advancing questionnaire state.
- Preferred sanitized browser transcript fallback when local STT output is unusable.
- Changed no-transcript private turns into a recoverable repeat prompt rather than an HTTP failure.
- Made `survey_id` optional via `ASK_SURVEY_ID=1`; the default spoken flow now starts from OA diagnosis like the reference call.
- Added narrow qualitative anchor parsing for `last_flare_pain_score`, including mild/no-sleep-impact style answers from the reference interview.
- Added smoke-test coverage for STT artifact rejection, browser fallback after bad STT, and a reference-style questionnaire replay.
- Added a short post-TTS delay before private-mode auto-listen to reduce assistant-audio tail capture.
- Expanded report quality metrics with aggregate STT, clinical-engine, and total turn latency fields.
- Added `scripts/check_v092_environment.py` for pre-demo checks of version metadata, STT model path, local Qwen configuration/health, and TTS sidecar health.

## Remaining Planned Changes

- Stronger semantic parsing for culturally fuzzy Chinese answers is promoted to v0.9.3; v0.9.2 remains a deterministic reliability baseline.
- Browser-level VAD thresholds may still need empirical tuning after real-device testing.

## Known Risks

- The current deterministic parser is keyword-based and cannot fully generalize to culturally fuzzy Chinese answers; this is the v0.9.3 target.
- True no-click full-duplex still requires always-on VAD/echo handling or a streaming voice runtime; v0.9.2 should not pretend otherwise.
