# v0.7.1

## Summary

Improves the v0.7 private pipeline with server-side voice synthesis support, model traceability, and targeted validation fixes from early demo conversations.

## Main Changes

- Added Fun-CosyVoice3 support as the preferred private server TTS path.
- Added model-event traces to private pipeline turns and final reports.
- Added trace metadata for local Qwen extraction and constrained reply rewriting.
- Exposed the latest private pipeline model event in `/api/private/turn`.
- Updated `assistant_version` in reports to `v0.7`.
- Documented server deployment settings for Fun-CosyVoice3.

## Demo-Failure Fixes

- Fixed identity parsing where a phone number followed by age could be merged into one long phone number.
- Added support for common pinyin-like comparison answer `chabuduo` as "same".
- Changed treatment-context handling so unclear answers such as `不懂` or "don't know" trigger clarification instead of being recorded as a treatment.

## Model Stack Target

```text
SenseVoiceSmall STT
  -> Qwen3-8B structured extractor / constrained verbalizer
  -> deterministic OA ConversationEngine
  -> Fun-CosyVoice3 TTS
```

## Known Limitations

- The conversation still feels too static because the report/evaluation loop does not yet convert poor conversations into structured improvement data.
- The current report is doctor-readable but not yet optimized for medical-research review, model debugging, or reinforcement-learning style preference data.
- Reply rewrite can make wording more natural, but it does not yet reason over full conversation quality.
- CosyVoice3 requires the server runtime and a valid prompt wav; browser TTS fallback remains necessary when server TTS fails.

## Next Version

v0.7.2 should focus on conversation documentation and feedback instrumentation:

- one report for doctors/researchers
- one audit trace for engineers
- one training/evaluation artifact for iterative AI improvement
