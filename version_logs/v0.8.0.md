# v0.8.0

Date: 2026-07-09

## Goal

Make the prototype look and feel like a polished patient-facing voice health assistant rather than an internal admin prototype, while preserving the existing deterministic OA clinical engine and private voice pipeline behavior.

## User-Facing Changes

- Rebuilt the main screen around a voice-call surface with a large assistant presence, clear call mode label, and prominent start/record controls.
- Added a privacy and safety trust strip that surfaces the private pipeline, clinical-rule control, doctor handoff, and research-prototype status.
- Replaced the plain state card with a protocol progress rail for identity, pain, function, treatment, side effects, red flags, and report completion.
- Reworked the transcript area into a cleaner call record with distinct assistant, patient, and system message styles.
- Added a formatted doctor-report preview so completed reports read like a clinical handoff by default, with raw JSON available through a toggle.
- Improved responsive behavior for tablet and mobile layouts.

## Backend Changes

- None. v0.8.0 is a frontend visual foundation update.

## Safety/Clinical-Rule Changes

- None. The deterministic clinical state machine, red-flag handling, report generation, and save endpoint remain unchanged.

## Known Limitations

- The voice visualization is state-driven, not real audio amplitude-driven.
- No browser screenshot audit has been performed yet in this pass.
