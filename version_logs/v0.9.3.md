# v0.9.3

Date: 2026-07-22

Status: Implemented baseline. Scope may still change; append later updates below rather than replacing this record.

## Goal

Improve the questionnaire engine's realtime handling of culturally fuzzy Chinese answers without letting wrong-bucket or conditional answers silently advance the formal questionnaire.

## Implemented Changes

- Added accepted-answer interpretation metadata with confidence, strategy, fuzzy, needs-review, and evidence fields.
- Kept deterministic step parsers as the clinical authority; fuzzy handling only annotates accepted answers and does not override unclear parser decisions.
- Added fuzzy diagnosis evidence handling for answers such as doctor-described osteoarthritis.
- Marked qualitative pain-score anchors and fuzzy improvement descriptions as auditable interpretations.
- Preserved clarification behavior for benefit-only adverse-reaction answers and conditional cost-dependent willingness answers.
- Added report quality metrics for fuzzy answer count, low-confidence answer count, interpretation review count, and per-answer interpretation flags.
- Added failure/training tags for low-confidence or review-needed semantic interpretations.
- Added v0.9.3 parser regression tests for fuzzy Chinese logic and wrong-input rejection.
- Added `scripts/check_v093_environment.py` and bumped report metadata to v0.9.3 / schema 0.9.3.

## Safety/Clinical Rule Notes

- v0.9.3 does not treat LLM or fuzzy parsing as a replacement for the questionnaire rulebook.
- Ambiguous, off-step, or conditional answers still trigger clarification instead of completing the field.
- Low-confidence accepted answers are visible in the generated report for human review.

## Known Limitations

- This is still a deterministic semantic layer, not a fully learned realtime dialogue understanding system.
- Fuzzy parser coverage is only as good as the regression set and observed stakeholder runs.
- True full-duplex interruption robustness still depends on browser VAD/echo behavior or a streaming voice runtime.
