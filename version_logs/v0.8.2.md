# v0.8.2

Date: 2026-07-09

## Goal

Make the completed doctor report feel like a usable clinical handoff rather than a raw JSON artifact, while preserving the single JSON report as the source of truth.

## User-Facing Changes

- Added a report summary band for follow-up priority, current pain, and red-flag status.
- Added a copy-report control with clipboard fallback.
- Expanded the formatted report preview into structured sections:
  - doctor handoff
  - patient identity
  - pain assessment
  - safety assessment
  - research review
  - conversation quality
  - limitations
- Added status chips for identity completeness, pain-score completeness, functional anchor completeness, red-flag status, and study usability.
- Improved the JSON/preview toggle so the formatted summary hides when raw JSON is shown.

## Backend Changes

- None. The existing `app/report.py` JSON payload remains the source of truth.

## Safety/Clinical-Rule Changes

- None. v0.8.2 changes only report presentation and copy/export affordances.

## Files Changed

- `static/index.html`
- `static/styles.css`
- `static/app.js`
- `version_logs/v0.8.2.md`
- `version_logs/README.md`
- `README.md`

## Tests Run

- JavaScript syntax check.
- Python compile check for core app modules.
- Python smoke test for the existing clinical flow.

## Known Limitations

- The formatted preview is client-side presentation of the saved JSON report; it does not create a separate clinician note artifact.
- Section labels are currently English even when the app is in Chinese because report keys are stable English clinical fields.

## Follow-Up Candidates

- Add clinician-review actions such as mark reviewed, needs callback, and export summary if those become part of the workflow.
- Add screenshot audits for the report panel at desktop and mobile sizes.
- Localize report section labels if stakeholders prefer a fully Chinese report UI.
