# v0.7

## Summary

Introduces the private explainable voice pipeline as the production architecture.

## Direction

v0.7 leaves Covo and GLM-4-Voice as research notes and uses a modular private stack:

```text
browser audio
  -> local STT
  -> local structured extractor
  -> deterministic OA rulebook
  -> constrained response wording
  -> local TTS
```

## Backend Changes

- Added `app/private_pipeline.py` to coordinate local STT, deterministic clinical handling, and per-turn latency metadata.
- Added `app/local_ai.py` for local OpenAI-compatible structured extraction and optional constrained wording rewrite.
- Wired `ConversationEngine` to use the local clinical AI adapter by default.
- Added `/api/private/start` for v0.7 private voice sessions.
- Added `/api/private/turn` for recorded patient turns with local STT and browser transcript fallback.
- Extended `/api/voice_status` with private pipeline and local AI readiness metadata.

## Frontend Changes

- The primary start button now launches the v0.7 private explainable pipeline.
- The second call button records one patient answer turn.
- Browser transcript capture remains a development fallback while local STT is unavailable or still loading.
- Assistant speech still uses server TTS first and browser TTS as fallback.

## Model Stack

- STT: SenseVoiceSmall through FunASR.
- Structured extraction: local Qwen-compatible model served at `/v1/chat/completions`.
- Clinical controller: deterministic `ConversationEngine`.
- TTS: Qwen3-TTS local path, then browser TTS fallback.

## Streaming Roadmap

v0.7 keeps turn recording for reliability but records timing metadata and defines the seams for:

- WebRTC audio frame streaming
- WebRTC VAD or Silero VAD endpointing
- partial ASR captions
- speculative extraction without state mutation
- streaming TTS
- automatic barge-in cancellation

## Research Notes

- Covo: archived as a half-duplex V2V experiment, not production path.
- GLM-4-Voice: future native V2V research candidate, not production path.
